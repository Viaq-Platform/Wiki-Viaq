# Offline Operations – Working Without International Internet

This section contains documentation and procedures for managing servers that have **no direct access to the international internet**. Here you will find methods to transfer Git repositories, Docker images, APT packages, and other dependencies using offline media or limited connections.

## Available Guides

- [Git Bundle – Transferring repositories using `git bundle`](git-bundle.md)
- *(more guides will be added as needed)*

## When to Use These Guides

- Your server is behind a firewall or has no outbound internet.
- You need to synchronise code, containers, or packages from a machine that does have internet (e.g., a local PC with VPN).
- You want to avoid opening the server to the internet for security reasons.
