# Viaq Handbook

Welcome to the **Viaq Handbook** – the central source of truth for organizational standards, technical guidelines, and internal documentation.

This handbook is maintained by the DevOps team and applies to all engineering and product teams across Viaq.

## 📖 Sections

- [Naming Conventions](naming-conventions.md) – unified rules for repositories, branches, projects, and artifacts.
- [Project Structure](project-structure.md) – standard layout for backend, frontend, and Dockerized applications.
- [Documentation Guide](documentation-guide.md) – best practices for writing and maintaining documentation in this wiki.
- [WikiViaq](WikiViaq/) – platform documentation for the WikiViaq project itself (setup, scripts, API tokens).

## 🎯 Principles

- **Consistency** – All teams follow the same patterns to reduce friction.
- **Simplicity** – Avoid over‑engineering; prefer clear, minimal solutions.
- **Automation** – Whenever possible, standards should be enforced by CI/CD pipelines or linters.
- **Living documents** – This handbook evolves with the organisation. Propose changes via pull requests.

## 🤝 Contribution

To propose a change, submit a pull request to the [WikiViaq repository](https://github.com/Viaq-Platform/Wiki-Viaq). All changes are reviewed by the DevOps lead.
